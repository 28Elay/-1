1. Работа над ошибками (Undo/Redo)

•   git commit --amend --no-edit
    Что это: «Блин, забыл файл» или «Опечатка в коммите».
    Суть: Вместо того чтобы делать второй коммит «Fix typo», ты просто вливаешь новые изменения в последний сделанный коммит. Выглядит так, будто ты всё сделал идеально с первого раза.

•   git reset --soft HEAD~1
    Что это: Откат коммита без потери кода.
    Суть: Ты сделал коммит, но передумал. Эта команда удаляет запись о коммите, но все твои изменения остаются в файлах. Они просто снова становятся «зелененькими» (в стейдже), готовыми к новой правке.

•   git reset --hard HEAD~1
    Что это: «Удали нафиг всё, что я только что сделал».
    Суть: Полный откат. Стирает и коммит, и все изменения в файлах до состояния предыдущего коммита. Используй осторожно — код исчезнет навсегда.

•   git revert <hash>
    Что это: Безопасный откат в общей ветке.
    Суть: Если ты уже запушил плохой код в общую ветку, reset использовать нельзя (история сломается). revert создает новый коммит, который делает всё ровно наоборот по сравнению с плохим коммитом. История чиста, проблема решена.

2. Выборочное воровство и прятки

•   git cherry-pick <hash>
    Что это: «Мне нужен только этот конкретный кусок из той ветки».
    Суть: Тебе не нужно мержить всю чужую ветку с её мусором. Ты просто тыкаешь пальцем в конкретный коммит из любой ветки, и Гит копирует его к тебе.

•   git stash / git stash pop
    Что это: «Карман» для кода.
    Суть: Ты пишешь фичу, и тут прилетает срочный баг. Чтобы не делать грязный «недокоммит», ты пишешь stash — все изменения прячутся в виртуальный карман, и файлы становятся чистыми. Пофиксил баг, вернулся и набрал stash pop — твои недописанные изменения вернулись.

3. Наведение порядка в истории

•   git rebase -i HEAD~3 (Интерактивный ребейс)
    Что это: Фотошоп для истории коммитов.
    Суть: Позволяет объединить (squash) пять мелких коммитов типа «fixed», «fixed again», «final fix» в один красивый и чистый. Можно переименовывать старые коммиты или удалять лишние.

•   git log --oneline --graph
    Что это: Понятная карта дорог.
    Суть: Рисует в консоли дерево веток. Сразу видно, кто куда влился и где история разошлась.

4. Продвинутый поиск (Когда всё горит)

•   git reflog
    Что это: Чёрный ящик самолета.
    Суть: Даже если ты сделал reset --hard и удалил важный коммит, Гит всё равно помнит его хеш в журнале reflog в течение пары недель. Это последняя надежда найти код, который ты сам случайно удалил.

•   git blame <file>
    Что это: «Кто это написал?!»
    Суть: Построчно показывает автора каждой строчки в файле. Незаменимо, чтобы найти того, кто внедрил баг три месяца назад.

•   git checkout <hash> -- <file>
    Что это: Вытащить файл из прошлого.
    Суть: Тебе не нужен весь старый коммит, тебе нужен один конкретный файл в том виде, в каком он был неделю назад. Эта команда заменит текущую версию файла на старую.

5. Поиск виноватого

•   git bisect
    Что это: Детектив.
    Суть: Если проект сломался, и ты не знаешь, какой из 50 коммитов это сделал. Ты говоришь: «Вот тут работало, а сейчас — нет». Гит начинает прыгать по коммитам методом деления пополам, заставляя тебя проверять код, пока не найдет того самого «киллера».
